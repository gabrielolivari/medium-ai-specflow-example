# Glossary

- **SDD (Spec-Driven Development):** Development approach where specifications drive implementation and validation.
- **Spec:** Functional source of truth for a use case.
- **Acceptance Criteria:** Verifiable conditions that define done.
- **Contract:** Formal interface definition (API schema, input/output, errors, events).
- **RFC (Request for Comments):** Proposal document evaluated before a decision.
- **ADR (Architecture Decision Record):** Final decision document with context and consequences.
- **Guardrail:** Non-negotiable safety/quality constraint.
- **Risk Matrix:** Probability-impact model used to prioritize risk mitigation.
- **Traceability:** Explicit links between spec, code, tests, and PRs.
